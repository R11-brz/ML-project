# Проект по машинному обучению: Прогнозирование рейтинга фильмов IMDB 
 Выполнили: студентки 3 курса ГУУ, группы ПМИ3-1, Гущина Ольга, Царапкина Ольга

 Цель проекта: Разработать модель машинного обучения для прогнозирования рейтинга фильмов на основе их характеристик (бюджет, жанр, язык, популярность и т.д.). Полученная модель может быть использована для оценки потенциального успеха нового фильма или для анализа факторов, влияющих на высокие оценки.

Источник данных: Данные были получены через API themoviedb.org (TMDb), который предоставляет информацию, аналогичную IMDB. Исходный файл — movies_ready_for_ml.csv.

1. Целевая переменная (Target Variable): vote_average — это средний пользовательский рейтинг фильма по шкале от 0 до 10. Является основной метрикой успеха фильма с точки зрения зрителей. Более высокий рейтинг означает, что фильм получил более высокие оценки от аудитории. Задача модели: Обучиться предсказывать это значение на основе других признаков.

2. Описание признаков (Features)
- vote_average - Целевая переменная. Средний рейтинг фильма (от 0 до 10).
- budget - Бюджет фильма в долларах. (Очищен от нулевых значений).
- runtime - Продолжительность фильма в минутах.
- popularity - Индекс популярности фильма на платформе TMDb, рассчитываемый на основе активности пользователей (просмотры, добавления в списки и т.д.).
- vote_count - Общее количество голосов пользователей, на основе которых рассчитан vote_average. Этот признак важен, так как рейтинг, основанный на малом количестве голосов, может быть нерепрезентативным.
- release_year - Год выхода фильма.
is_Action, is_Comedy, ... , is_Animation - Кодировка жанров. Каждый столбец соответствует одному жанру. Значение 1 означает, что фильм относится к этому жанру (фильм может иметь несколько жанров).
- lang_en, lang_ru, ... , lang_other - Кодировка основного языка фильма. Значение 1 означает, что фильм снят на соответствующем языке. lang_other — для всех остальных языков.
- budget_log - Логарифм бюджета ( np.log1p(budget)). Создан для нормализации сильно скошенного распределения бюджетов.
- vote_count_log - Логарифм количества голосов (np.log1p(vote_count)). Помогает модели лучше воспринимать очень большие различия в этой метрике.
- budget_per_popularity - Отношение бюджета к популярности ( budget / (popularity + 1)). Добавлен для создания нелинейной взаимосвязи между этими двумя признаками.

3. Краткое описание этапов работы и полученных результатов.

Загрузка и первичный анализ данных: Изучена структура датасета, выявлены пропуски и типы данных.

Предобработка данных:
- Удалены строки с пропусками.
- Созданы новые признаки: budget_log, vote_count_log, budget_per_popularity.
- Проведен анализ и обработка выбросов.

Исследовательский анализ данных (EDA):
- Проанализированы распределения целевой переменной и ключевых признаков.
- Построена матрица корреляций, выявлены факторы, наиболее сильно влияющие на рейтинг (например, vote_count, runtime, принадлежность к жанру "Драма").

Подготовка к моделированию: Данные разделены на обучающую и тестовую выборки (80/20), проведена стандартизация числовых признаков.

Обучение и оценка моделей: Обучены и сравнены несколько моделей. Лучшая модель: Gradient Boosting.

Ключевые метрики на тестовой выборке: R² = 0.5692, MAE = 0.3606.

Визуализация результатов и анализ важности признаков. Наиболее важными факторами для предсказания рейтинга оказались: соотношение budget_per_popularity, runtime, is_Drama и vote_count.

Оптимизация гиперпараметров (Grid Search): Позволила незначительно улучшить качество модели (R² = 0.5836).

 На основе важности признаков мы можем сделать такие выводы:

- *Бюджет в соотношении с популярностью* — самый важный фактор (почти 19%). Это означает, что не просто большой бюджет важен сам по себе, а его эффективность: фильмы, которые окупаются в популярности, получают более высокие оценки.

- *Хронометраж* — второй по значимости фактор (~14%). Слишком короткие фильмы (<90 минут) часто воспринимаются как поверхностные, слишком длинные (>150 минут) рискуют утомить зрителя. Оптимальная длительность, судя по данным — около 100-120 минут.

- *Жанр драмы* — наиболее привлекательный жанр (почти 11% важности). Драматические фильмы традиционно получают более высокие оценки критиков и зрителей. Интересно, что жанры боевик, комедия и ужасы оказались наименее значимыми для высокого рейтинга.

- *Количество голосов* — еще один важный фактор (около 10%). Это говорит о том, что чем больше людей оценили фильм, тем более "усредненный" и репрезентативный у него рейтинг, и модель это учитывает.

Общий вывод: Модель Gradient Boosting продемонстрировала лучшую способность к обобщению, предсказывая рейтинг со средней ошибкой около ±0.36 балла. Это говорит о том, что на основе имеющихся данных (бюджет, жанр, популярность, год выпуска и т.д.) можно с достаточной точностью оценить будущую оценку фильма зрителями. Наибольшее влияние на рейтинг оказывают драматические фильмы, а также факторы, связанные с вовлеченностью аудитории (vote_count) и производственными затратами (budget).

Кинокомпания, снимающая новый фильм, может заложить в модель его параметры (бюджет, хронометраж, жанр, предполагаемую популярность) и получить прогноз ожидаемого рейтинга.
